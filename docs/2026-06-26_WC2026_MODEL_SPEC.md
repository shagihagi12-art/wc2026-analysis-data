# WC2026 公式状態・ルート参照仕様

## 目的

2026 FIFAワールドカップの**現時点の確定スコア、順位、3位比較、決勝トーナメントの公式ルート**を扱う正本です。

この正本は「現況を正しく再計算するための入力層」です。勝率・モンテカルロ・戦力評価の出力は含めません。

## データ時点

- スナップショット: `2026-06-26T19:16:44+09:00`
- 取得元: `ChatGPT soccer_schedule_summary_source / WorldCup / generated 2026-06-26T10:16:44Z`
- 対象: グループA〜Fは最終節まで反映済み。G〜Lは最終節前。
- 日本時間の最終節: G/H/Iは2026-06-27、J/K/Lは2026-06-28

## 事実と推定の境界

### 事実
- 試合スコア
- 現時点の勝点・得失点差・総得点
- 最終節のカード
- FIFA規定上の順位比較順
- Annex Cの495通りの対応表

### 未確定
- G〜L組の最終順位
- 上位8の3位通過チーム
- 3位通過チームの最終対戦カード
- 同記録時のフェアプレー比較に必要な最終チームコンダクト値

### 推定としてのみ扱うもの
- 勝率、進出確率、モンテカルロ
- Readiness、Elo、xG、移動負荷による優劣
- 3位通過の確率分布

推定を出す場合は、必ずこの公式状態を固定入力にし、別ファイルとして日付付きで出力します。

## FIFA規定の扱い

### グループ内順位
勝点が並んだ場合は、当該チーム間の比較を優先します。  
その後に全試合の得失点差、総得点、フェアプレー、FIFAランキングへ進みます。

### 3位比較
上位8の3位チームは、全試合の

1. 勝点
2. 得失点差
3. 総得点
4. フェアプレー
5. FIFAランキング

で比較します。

### Annex C
3位通過8組が確定した後、R32の3位対戦相手はAnnex Cの495通りに厳密に接続します。手作業で対戦先を推定しません。

## フェアプレーの扱い

この正本では、最終フェアプレー値を未検証のまま順位算定に入れません。  
勝点・得失点差・総得点がすべて同じ場合だけ、FIFA Match Centreまたは公式試合記録からチームコンダクトを再確認して更新します。

## 日本代表の現時点の対戦ルート

- R16: 上記の勝者 vs コートジボワール vs I組2位の勝者
- I組2位:
  - フランスがノルウェーに勝利または引き分け → ノルウェー
  - ノルウェーがフランスに勝利 → フランス

## 更新手順

1. `WC2026_MatchFacts.csv` に最終スコアを反映
2. `WC2026_OfficialState.csv` と `WC2026_BestThird_Tracker.csv` を再計算
3. 同値がある場合はフェアプレーを公式記録で確認
4. 上位8の3位組セットを確定
5. Annex CでR32配置を確定
6. 確率モデルを使う場合だけ、別日付のモデル出力を作成
